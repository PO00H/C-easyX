## 盲剑客：《白夜瞬闪》20天开发计划

### 一、文件管理原则

1. **单一职责**：一个文件尽量只负责一个模块，不把输入、渲染、逻辑、资源加载混在一起。
2. **先建骨架，再补功能**：先把类、头文件、接口定下来，再逐步实现内容。
3. **公共内容集中管理**：常量、枚举、基础结构体统一放在 `common.h`。
4. **资源集中加载**：图片、音效、地图文件统一走资源管理器，避免重复加载。
5. **主循环统一调度**：所有模块最终都由 `Game` 统一更新和绘制。

---

### 二、建议的工程文件结构

#### 1. 第一版最小文件集

- `main.cpp`：程序入口，只负责启动游戏
- `game.h / game.cpp`：主循环、场景切换、初始化与退出
- `player.h / player.cpp`：玩家移动、面向、蓄力、突进
- `entity.h / entity.cpp`：实体基类，提供位置、碰撞体、生存状态
- `common.h`：公共枚举、结构体、常量

#### 2. 第一阶段建议补充的模块

- `map.h / map.cpp`：地图读取、墙体生成、出生点管理
- `input.h / input.cpp`：键盘与鼠标输入封装
- `resource_manager.h / resource_manager.cpp`：图片、音效、字体加载
- `effect_manager.h / effect_manager.cpp`：顿帧、震动、粒子等效果状态
- `enemy.h / enemy.cpp`：敌人基类与后续 AI 扩展接口

### 四、各文件职责说明

#### `main.cpp`

- 创建游戏对象
- 调用初始化
- 进入主循环
- 程序退出时释放资源

**原则**：不要把具体玩法写进 `main.cpp`。

#### `game.h / game.cpp`

- 控制游戏整体流程
- 管理当前场景
- 调用 `Update()` 和 `Draw()`
- 处理菜单、游戏、暂停等状态切换

**原则**：它是总控层，不直接写玩家细节或敌人 AI。

#### `player.h / player.cpp`

- 实现 WASD 移动
- 实现鼠标方向朝向
- 实现蓄力与突进状态
- 预留波纹探测接口

**第一天目标**：先做“能移动的玩家”。

#### `entity.h / entity.cpp`

- 定义所有实体的公共属性
- 包含位置、尺寸、碰撞矩形、生存状态
- 提供基础更新和绘制接口

**作用**：后续玩家、敌人、灯具都可以继承它。

#### `map.h / map.cpp`

- 读取字符地图
- 生成墙体碰撞体
- 提供出生点
- 后续扩展灯具、敌人、障碍物

**建议地图格式**：

- `#`：墙壁
- `.`：空地
- `P`：玩家出生点
- `E`：敌人出生点
- `L`：灯具
- `B`：障碍物

#### `input.h / input.cpp`

- 统一处理键盘输入
- 统一处理鼠标输入
- 区分游戏内和菜单中的输入逻辑

**作用**：避免以后每个类都直接写按键判断。

#### `resource_manager.h / resource_manager.cpp`

- 集中加载图片
- 集中加载音效
- 集中管理资源路径

**原则**：不要在业务逻辑里到处 `loadimage`。

#### `effect_manager.h / effect_manager.cpp`

- 管理顿帧时间
- 管理屏幕震动偏移
- 管理粒子和血迹数据

**原则**：只保存效果状态，不直接负责实体绘制。

#### `enemy.h / enemy.cpp`

- 定义敌人基类
- 预留视野、巡逻、听觉反应接口
- 后续扩展近战步卒、远程弓手

**第一天可以先只建空壳。**

### 五、第一天的具体开发顺序

#### 第 1 步：建立工程骨架

1. 新建 `common.h`
2. 新建 `entity.h / entity.cpp`
3. 新建 `game.h / game.cpp`
4. 新建 `player.h / player.cpp`
5. 确认程序可以正常启动窗口并进入主循环

#### 第 2 步：先跑通玩家基础移动

1. 玩家可以使用 WASD 移动
2. 玩家有基础坐标和碰撞矩形
3. 玩家能根据鼠标指向更新朝向

#### 第 3 步：预留后续系统接口

1. 新建 `map.h / map.cpp` 空壳
2. 新建 `input.h / input.cpp` 空壳
3. 新建 `resource_manager.h / resource_manager.cpp` 空壳
4. 新建 `effect_manager.h / effect_manager.cpp` 空壳
5. 新建 `enemy.h / enemy.cpp` 空壳

### 六、20天开发节奏建议

#### 第一阶段：工程骨架与手感原型（第1~5天）

- **第1天**：目录结构、基础类、玩家移动、主循环
- **第2天**：输入系统整理，统一键鼠处理
- **第3天**：蓄力与突进原型
- **第4天**：顿帧、震动、基础粒子
- **第5天**：音效接入，完成战斗原型[] 推迟

#### 第二阶段：核心机制与第一关（第6~10天）

- **第6天**：波纹探测
- **第7天**：记忆迷雾
- **第8天**：地图读取和墙体生成
- **第9天**：第一关地图
- **第10天**：波纹表现优化

#### 第三阶段：敌人与环境交互（第11~15天）

- **第11天**：敌人基类和视野系统
- **第12天**：近战敌人与灯具破坏
- **第13天**：第二关地图与障碍物
- **第14天**：远程敌人与警戒逻辑
- **第15天**：三关切换流程

#### 第四阶段：包装与收尾（第16~20天）

- **第16天**：血迹粒子与打击反馈
- **第17天**：菜单和过场
- **第18天**：数值调整和平衡测试
- **第19天**：Bug 修复和代码整理
- **第20天**：打包导出和最终检查

### 七、今天最适合完成的任务

如果是第一天，建议只完成下面这些内容：

1. 把 `main.cpp / game.h / game.cpp / player.h / player.cpp` 跑通
2. 增加 `common.h` 和 `entity.h`
3. 让玩家能在窗口里移动
4. 为后续的地图、敌人、资源、特效留好接口

**第一天目标不是功能完整，而是结构正确。**

---

### 八、后续扩展建议

- 如果后面文件增多，可以继续拆成更细模块，例如 `player_movement`、`player_attack`、`enemy_melee`、`enemy_archer`
- 如果资源变多，尽早把 `assets/` 目录固定下来
- 如果地图复杂，优先统一用字符地图或文本地图，不要一开始就依赖复杂编辑器

1. `main.cpp`
2. `common.h`
